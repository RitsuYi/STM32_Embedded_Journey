# 蓝牙调试说明

## 1. 基本信息

- 蓝牙串口固定为 `USART3`
- 引脚连接：
  - `PD8 -> BT_RX`
  - `PD9 <- BT_TX`
- 当前波特率：`9600`
- 数据格式：`8N1`

本工程已经按蓝牙串口小程序的数据包格式做了适配，当前重点支持：

- `绘图` 页面：显示左前轮电机的设定值、实际值、PWM 输出值
- `滑杆` 页面：在线修改速度环 PID 的 `Kp / Ki / Kd`，以及前进目标速度 `T`

蓝牙调参视频教程可参考：

- `https://www.bilibili.com/video/BV1aDndzEEk1`

## 2. 当前适配范围

### 2.1 绘图模式

单片机会持续回传标准绘图数据包：

```text
[plot,左前轮设定值CPS,左前轮实际值CPS,左前轮PWM]
```

说明：

- 只发送左前轮 3 条曲线
- 第 1 条线：速度环设定值
- 第 2 条线：编码器实际测速值
- 第 3 条线：PID 输出后的 PWM 命令值
- 当前连续回传周期由 `BOARD_BLUETOOTH_TELEMETRY_MS` 决定，默认是 `100ms`

示例：

```text
[plot,1200,1136,428]
```

### 2.2 滑杆模式

单片机接收标准滑杆数据包：

```text
[slider,P,0.01]
[slider,I,0.01]
[slider,D,0.01]
[slider,T,300]
```

对应关系：

- `[slider,P,x]`：设置速度环 `Kp = x`
- `[slider,I,x]`：设置速度环 `Ki = x`
- `[slider,D,x]`：设置速度环 `Kd = x`
- `[slider,T,x]`：设置前进模式基础目标速度 `x`

补充说明：

- 也兼容滑杆名称写成 `KP / KI / KD / TARGET`
- 也兼容小程序短格式：

```text
[s,P,0.01]
[s,I,0.01]
[s,D,0.01]
[s,T,300]
```

- 当前版本只保留速度环 PID
- 航向环 / 角度环当前已关闭，不参与直行纠偏
- `T` 的最大值受 `BOARD_CAR_MOVE_TARGET_SPEED_MAX_RPM` 限制，超出后会自动截断到 `BoardConfig.h` 设定上限

## 3. 小程序使用建议

### 3.1 绘图页面配置

建议在小程序的“绘图”页面中按顺序观察 3 条曲线：

1. 左前轮设定值
2. 左前轮实际值
3. 左前轮 PWM 输出

如果绘图页面没有数据，先检查：

1. 是否已经连接蓝牙
2. 是否已经发送 `[TEL,ON]`
3. 蓝牙模块是否确实在闭环运行

### 3.2 滑杆页面配置

建议建立 4 个滑杆，名称分别为：

- `P`
- `I`
- `D`
- `T`

推荐做法：

- 步距设置成较小的小数，例如 `0.01`
- 先只调 `P`
- 再补少量 `I`
- 最后按需要加入少量 `D`
- `T` 建议使用整数步距，范围上限直接按 `BOARD_CAR_MOVE_TARGET_SPEED_MAX_RPM` 配置

## 4. 常用蓝牙命令

除了小程序的 `plot / slider` 数据包外，发送区仍可直接发送下面这些文本命令。

### 4.1 查询速度环 PID

```text
[PID,GET]
```

返回示例：

```text
[PID,SPEED,0.1000,0.0500,0.0100,1200.00,1000.00]
```

字段顺序：

```text
[PID,SPEED,Kp,Ki,Kd,积分限幅,输出限幅]
```

说明：

- 现在只返回速度环 PID
- 不再通过蓝牙返回或设置航向环 PID

### 4.2 直接修改速度环 PID

```text
[PID,SET,SPEED,0.12,0.05,0.01]
```

这是文本命令方式，和滑杆方式作用相同，都是修改速度环 PID。

### 4.3 修改速度环积分/输出限幅

```text
[PID,LIMIT,SPEED,1200,700]
```

字段说明：

- 第 4 个参数：积分限幅
- 第 5 个参数：输出限幅

### 4.4 开启、关闭、单次获取绘图数据

开启连续回传：

```text
[TEL,ON]
```

关闭连续回传：

```text
[TEL,OFF]
```

单次获取一帧绘图数据：

```text
[TEL,GET]
```

回传格式固定为：

```text
[plot,左前轮设定值CPS,左前轮实际值CPS,左前轮PWM]
```

### 4.5 查询控制配置

```text
[CFG,GET]
```

返回格式：

```text
[CFG,基础直线目标速度CPS,转向目标速度CPS,速度环输出限幅PWM,遥测周期ms]
```

### 4.6 修改速度目标和转向目标

基础直线目标速度：

```text
[CTRL,SPEED,1200]
[slider,T,300]
```

转向目标速度：

```text
[CTRL,TURN,900]
```

速度环输出限幅：

```text
[CTRL,PWM,700]
```

### 4.7 查询或设置小车状态

查询：

```text
[STATE,GET]
```

设置：

```text
[STATE,SET,FORWARD]
[STATE,SET,BACKWARD]
[STATE,SET,LEFT]
[STATE,SET,RIGHT]
[STATE,SET,STOP]
```

如果是小程序“响应按键”，也可以直接使用更短的两段式命令：

```text
[MODE,STOP]
[MODE,FORWARD]
[MODE,BACKWARD]
```

建议把 3 个响应按键名称分别配置为：

- `停止模式` -> `[MODE,STOP]`
- `前进模式` -> `[MODE,FORWARD]`
- `后退模式` -> `[MODE,BACKWARD]`

### 4.8 查询 MPU6050 状态

```text
[MPU,GET]
```

重新做陀螺仪零漂标定：

```text
[MPU,ZERO]
```

注意：

- 发送 `[MPU,ZERO]` 时小车必须保持静止

## 5. 典型调试流程

1. 小车上电后先不要晃动，等 MPU6050 完成初始化
2. 连接蓝牙后先发 `[MPU,GET]`，确认模块在线
3. 再发 `[PID,GET]`，确认当前速度环参数
4. 发 `[TEL,ON]`，打开绘图连续回传
5. 在小程序“绘图”页面观察左前轮设定值、实际值和 PWM
6. 在小程序“滑杆”页面用 `P / I / D / T` 四个滑杆在线调参

## 6. 调参建议

### 6.1 速度环

推荐顺序：

1. 先调 `Kp`
2. 再补少量 `Ki`
3. 最后再少量加入 `Kd`

现象参考：

- 跟随慢：适当增大 `Kp`
- 长时间达不到目标值：适当增大 `Ki`
- 输出抖动明显：减小 `Kd`，必要时同时减小 `Kp`
- PWM 经常打满：先降低目标速度，再判断是否需要放宽输出限幅

### 6.2 说明

- 当前蓝牙调试只围绕速度环展开
- 航向环 / 角度环当前未启用，直线效果依赖四路速度闭环
