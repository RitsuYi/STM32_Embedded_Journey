# 当前进度总结

## 1. 当前目标

当前已经从“上电固定 PWM 直行”推进到“上电自动进入双轮速度 PID 直行控制”，重点解决：

- 小车放在地上时动力偏弱、起步困难
- 左右轮本体差异导致的直行不稳
- `TIM1` 同时做 PWM 和控制中断的资源冲突

## 2. 已完成修改

### 2.1 主控制逻辑已切换为双轮独立速度 PID

修改文件：

- `User/main.c`

当前逻辑：

- 上电初始化 `Motor / Encoder / PID / Timer`
- 延时 `500ms`
- 置 `RunFlag = 1`
- 由 `TIM4_IRQHandler()` 每 `10ms` 执行一次速度闭环

当前结构：

```c
LeftPID.Target  = TARGET_SPEED_COUNT;
LeftPID.Actual  = leftSpeed;
PID_Update(&LeftPID);

RightPID.Target = TARGET_SPEED_COUNT;
RightPID.Actual = rightSpeed;
PID_Update(&RightPID);
```

### 2.2 控制定时器已从 TIM1 改为 TIM4

修改文件：

- `SYSTEM/Timer.c`

已完成：

- 原本控制中断使用 `TIM1`
- 现在改为 `TIM4`
- 避免与 `Hardware/PWM.c` 中的 `TIM1 PWM` 冲突

这一步很关键，否则 PID 控制节拍和 PWM 基准会互相影响。

### 2.3 PID 结构已补充积分限幅和基础抗饱和

修改文件：

- `User/PID.h`
- `User/PID.c`

新增字段：

- `IntegralMax`
- `IntegralMin`
- `LastOut`

已完成能力：

- 积分限幅
- 输出限幅
- 输出饱和时的简单抗积分累积

## 3. 当前参数

文件：

- `User/main.c`

当前关键参数如下：

```c
#define SPEED_LOOP_MS        10U
#define TARGET_SPEED_COUNT   18.0f

#define LEFT_START_PWM       20
#define RIGHT_START_PWM      18
#define LEFT_BIAS_PWM        4
#define RIGHT_BIAS_PWM       0

#define PID_OUT_MAX          80.0f
#define PID_OUT_MIN          0.0f
#define PID_INT_MAX          120.0f
#define PID_INT_MIN          -120.0f
```

当前 PID 参数：

```c
LeftPID.Kp  = 3.0f;
LeftPID.Ki  = 0.35f;
LeftPID.Kd  = 0.0f;

RightPID.Kp = 3.0f;
RightPID.Ki = 0.35f;
RightPID.Kd = 0.0f;
```

说明：

- `TARGET_SPEED_COUNT` 是当前每 `10ms` 期望编码器计数量级
- `LEFT_START_PWM / RIGHT_START_PWM` 用来补偿电机起步死区
- `LEFT_BIAS_PWM` 用来补偿左轮偏慢

## 4. 当前代码状态

目前版本特点：

- 不再使用之前的开环测试主函数
- 不再使用循迹主循环
- 不依赖 `Serial_Init()`
- 也没有重新接回 OLED 调试显示
- 当前版本聚焦于“先让车在地上能跑起来并保持更直”

当前中断执行逻辑：

1. `TIM4` 每 `1ms` 进入一次中断
2. 内部分频到 `10ms`
3. 读取左右编码器计数
4. 左右轮分别做 PID
5. 叠加起步补偿和左轮偏置
6. 输出到两个电机

## 6. 当前已知限制

当前版本仍有几个限制：

### 6.1 目前只适合“前进直行”

当前速度取值使用的是绝对值：

```c
leftSpeed = AbsCountToSpeed(Encoder_Get(1));
rightSpeed = AbsCountToSpeed(Encoder_Get(2));
```

因此：

- 当前适合只做前进闭环
- 还不适合直接扩展成前进/后退通用控制

### 6.2 参数仍需现场调试

虽然已经加了 PID 和起步补偿，但不同小车之间仍可能存在：

- 电机死区不同
- 左右轮机械阻力不同
- 编码器方向或量级不完全一致

所以现在这套参数更像“可跑起来的第一版”，不是最终定型值。

### 6.3 编码器方向尚未在这版中再次复核

当前版本为了先让小车能跑起来，直接对编码器计数取了绝对值。

这意味着：

- 如果后续要做后退、转向、完整速度控制
- 还需要回头重新确认编码器正负方向定义

## 7. 下一步建议

建议按这个顺序继续：

1. 先实车测试地面起步效果，观察是否已经能稳定前进
2. 如果还是偏弱，先调大 `TARGET_SPEED_COUNT`
3. 如果还是起步困难，再调大 `LEFT_START_PWM / RIGHT_START_PWM`
4. 如果还是左轮偏慢，再增大 `LEFT_BIAS_PWM`
5. 小车能稳定直行后，再补 OLED 或蓝牙调参
6. 最后再做编码器方向统一、后退支持和更完整的速度控制结构
