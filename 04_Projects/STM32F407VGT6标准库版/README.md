# STM32F407 L150Pro 小车工程说明

## 1. 当前目录分析

当前 `USER` 目录是 Keil 工程入口目录，真正的功能代码分散在上一级的多个模块目录中：

| 目录 | 作用 |
| --- | --- |
| `USER` | 工程入口、启动文件、中断文件、`main.c` |
| `../BALANCE` | 小车控制主逻辑、运动学、车型选择、系统初始化、OLED/APP显示 |
| `../HARDWARE` | ADC、CAN、编码器、电机 PWM、OLED、PS2、串口、定时器、MPU6050 |
| `../SYSTEM` | 延时、位带 GPIO、`printf` 串口重定向 |
| `../FWLIB` | STM32F4 标准外设库 |

主程序非常简洁：

- `main.c` 先调用 `systemInit()` 做全部硬件初始化。
- `while(1)` 中循环执行：
  - `show_task()`：电压采样、OLED 显示、APP 数据输出
  - `PS2_Read()`：读取 PS2 手柄
  - `data_task()`：按 10Hz 发送串口/CAN 状态数据
- 真正的运动控制闭环在 `TIM7_IRQHandler()` 中以 10ms 一次执行，也就是 100Hz 控制频率。

## 2. 上电初始化流程

`systemInit()` 的初始化顺序如下：

1. 中断分组、延时模块
2. LED、蜂鸣器、使能开关、OLED、按键
3. 串口 1/2/3/5
4. ADC：车型电位器、电池电压
5. CAN1
6. 根据电位器档位选择车型参数
7. 四路编码器 TIM2/TIM3/TIM4/TIM5
8. 舵机 PWM（TIM12）
9. 航模遥控输入捕获（TIM8）
10. 四路电机 PWM（TIM1/TIM9/TIM10/TIM11）
11. MPU6050 软件 I2C 与传感器初始化
12. PS2 手柄初始化
13. TIM7 10ms 控制中断

## 3. 引脚连接总表

### 3.1 基础 IO

| 功能 | 引脚 | 外设/模式 | 说明 |
| --- | --- | --- | --- |
| LED | `PA12` | GPIO 输出 | 板载状态灯 |
| 蜂鸣器 | `PA8` | GPIO 输出 | 蜂鸣器控制 |
| 用户按键 | `PE0` | GPIO 输入上拉 | 单击/双击/长按检测 |
| 使能开关 | `PD3` | GPIO 输入上拉 | `EN=1` 才允许输出 PWM |

### 3.2 模拟输入

| 功能 | 引脚 | ADC 通道 | 说明 |
| --- | --- | --- | --- |
| 车型选择电位器 | `PB1` | `ADC1_IN9` | 上电时根据档位选择车型 |
| 电池电压采样 | `PB0` | `ADC2_IN8` | 电池电压检测，低于约 10V 会关断输出 |

车型电位器从低到高对应：

| 档位序号 | 车型 |
| --- | --- |
| 0 | 麦克纳姆轮 `Mec_Car` |
| 1 | 三全向轮 `Omni_Car` |
| 2 | 阿克曼 `Akm_Car` |
| 3 | 两轮差速 `Diff_Car` |
| 4 | 四驱 `FourWheel_Car` |
| 5 | 履带/坦克 `Tank_Car` |

### 3.3 串口与 CAN

| 接口 | 引脚 | 参数 | 当前用途 |
| --- | --- | --- | --- |
| `USART1` | `PA9 TX`, `PA10 RX` | `115200 8N1` | 可用于 ROS / 上位机控制 |
| `USART2` | `PD5 TX`, `PD6 RX` | `9600 8N1` | 蓝牙 APP；`printf` 也重定向到这里 |
| `USART3` | `PD8 TX`, `PD9 RX` | `115200 8N1` | 默认 ROS / 上位机控制口 |
| `UART5` | `PC12 TX`, `PD2 RX` | `115200 8N1` | 备用上位机控制口 |
| `CAN1` | `PD0 RX`, `PD1 TX` | 约 `1Mbps` | CAN 控制与状态回传 |

### 3.4 OLED 与传感器

| 功能 | 引脚 | 模式 | 说明 |
| --- | --- | --- | --- |
| OLED `DC/RS` | `PD11` | GPIO 输出 | 软件 SPI 控制线 |
| OLED `RST` | `PD12` | GPIO 输出 | 复位 |
| OLED `SDIN` | `PD13` | GPIO 输出 | 数据线 |
| OLED `SCLK` | `PD14` | GPIO 输出 | 时钟线 |
| MPU6050 `SCL` | `PB10` | 软件 I2C | 时钟 |
| MPU6050 `SDA` | `PB11` | 软件 I2C | 数据 |

说明：

- OLED 这里使用的是软件 SPI/三线写入方式，代码里没有片选脚。
- MPU6050 使用软件模拟 I2C，不占用 STM32 的硬件 I2C 外设。

### 3.5 PS2 手柄接口

| PS2 信号 | 引脚 | 方向 | 说明 |
| --- | --- | --- | --- |
| `CLK` | `PE8` | 输出 | 时钟 |
| `CS` | `PE10` | 输出 | 片选 |
| `CMD/DO` | `PE12` | 输出 | 主机发命令 |
| `DAT/DI` | `PE15` | 输入 | 手柄返回数据 |

### 3.6 航模遥控接收机输入

当前固件把 `TIM8` 配成输入捕获，用来接收 4 路 PWM 航模信号：

| 遥控通道 | 引脚 | 定时器通道 |
| --- | --- | --- |
| CH1 | `PC6` | `TIM8_CH1` |
| CH2 | `PC7` | `TIM8_CH2` |
| CH3 | `PC8` | `TIM8_CH3` |
| CH4 | `PC9` | `TIM8_CH4` |

说明：

- 当前固件使用的是 `TIM8_Cap_Init()`，所以 `PC6~PC9` 现在是遥控接收输入。
- `timer.c` 里也保留了 `TIM8_SERVO_Init()`，如果改成舵机 PWM，这 4 个引脚会被复用成舵机输出，不能和航模接收同时使用。

### 3.7 电机 PWM 输出

每路电机都用两路 PWM 控制 H 桥正反转：

| 电机逻辑通道 | 引脚 1 | 定时器 | 引脚 2 | 定时器 | 说明 |
| --- | --- | --- | --- | --- | --- |
| Motor A | `PB8` | `TIM10_CH1` | `PB9` | `TIM11_CH1` | 差速/阿克曼里通常是左轮 |
| Motor B | `PE5` | `TIM9_CH1` | `PE6` | `TIM9_CH2` | 差速/阿克曼里通常是右轮 |
| Motor C | `PE11` | `TIM1_CH2` | `PE9` | `TIM1_CH1` | 四驱/麦轮/全向扩展通道 |
| Motor D | `PE14` | `TIM1_CH4` | `PE13` | `TIM1_CH3` | 四驱/麦轮/全向扩展通道 |

### 3.8 舵机 PWM

| 功能 | 引脚 | 定时器 | 说明 |
| --- | --- | --- | --- |
| Servo CH1 | `PB14` | `TIM12_CH1` | 已初始化，但当前控制逻辑未直接使用 |
| Servo CH2 | `PB15` | `TIM12_CH2` | 当前阿克曼转向舵机实际使用引脚 |

说明：

- `motor.h` 中 `Servo_PWM` 直接映射到 `TIM12->CCR2`，因此当前阿克曼舵机请按 `PB15` 接。

### 3.9 编码器接口

| 编码器接口 | 引脚 | 定时器 |
| --- | --- | --- |
| Encoder 1 | `PA15`, `PB3` | `TIM2` |
| Encoder 2 | `PB4`, `PB5` | `TIM3` |
| Encoder 3 | `PB6`, `PB7` | `TIM4` |
| Encoder 4 | `PA0`, `PA1` | `TIM5` |

重要说明：

- `system.c` 中的注释写的是：
  - `TIM2` 读电机 C
  - `TIM3` 读电机 D
  - `TIM4` 读电机 B
  - `TIM5` 读电机 A
- 但 `balance.c` 的 `Get_Velocity_Form_Encoder()` 又把 `TIM2/3/4/5` 顺序读入 `OriginalEncoder.A/B/C/D`。
- 也就是说，源码里的“编码器接口命名”和“电机逻辑通道命名”存在不完全一致的地方。实际接线时，建议优先以原理图/板子丝印/上电实测转向为准。

## 4. 车型与逻辑通道关系

不同车型会复用同一套 PWM/编码器资源：

| 车型 | 实际使用的执行器 |
| --- | --- |
| 麦克纳姆 | `Motor A/B/C/D` 四轮 |
| 三全向轮 | `Motor A/B/C` 三轮，`Motor D` 不用 |
| 阿克曼 | `Motor A/B` + `Servo(PB15)` |
| 两轮差速 | `Motor A/B` |
| 四驱 | `Motor A/B/C/D` |
| 履带车 | `Motor A/B` |

从显示逻辑可以看出：

- 差速/履带/阿克曼模式下，`Motor A` 被当作左侧，`Motor B` 被当作右侧。
- 四驱模式下，`A/B` 是一侧，`C/D` 是另一侧。

## 5. 使用方法

### 5.1 编译与烧录

1. 用 Keil 打开根目录 `Project.uvprojx`。
2. 编译后下载到板子。
3. 上电前先把电位器拨到需要的车型档位。
4. 接好电机、编码器、传感器、控制接口。
5. 打开使能开关，确保 `PD3` 读到高电平。

### 5.2 上电后的工作过程

1. 程序读取电位器，确定当前底盘类型。
2. 初始化编码器、PWM、IMU、串口、CAN、OLED、PS2、遥控输入。
3. `TIM7` 以 100Hz 运行控制闭环。
4. 主循环持续做显示、手柄读取、10Hz 数据上报。

### 5.3 控制方式

| 控制方式 | 接口 | 进入条件 |
| --- | --- | --- |
| ROS 默认控制 | `USART3` | 默认状态下即使用该控制通道 |
| 上位机控制 1 | `USART1` | 收到合法 11 字节控制帧后进入 |
| 上位机控制 2 | `UART5` | 收到合法 11 字节控制帧后进入，但当前代码有已知问题，见“源码备注” |
| 蓝牙 APP | `USART2` | APP 前进命令触发后进入 |
| PS2 手柄 | `PE8/10/12/15` | 按 `Start` 后再推动摇杆进入 |
| 航模遥控 | `PC6~PC9` | `CH2 > 1600` 时进入 |
| CAN 控制 | `CAN1` | 收到 `ID=0x181` 的控制帧后进入 |

OLED 最后一行会显示当前控制源：

- `ROS`
- `USART`
- `APP`
- `PS2`
- `R-C`
- `CAN`

### 5.4 允许输出的条件

只有同时满足下面条件时，电机 PWM 才会真正输出：

1. 电池电压不低于约 `10V`
2. 使能开关 `PD3=1`
3. `Flag_Stop==0`

否则程序会主动把 4 路电机 PWM 清零。

## 6. 串口/CAN 协议摘要

### 6.1 USART1 / USART3 / UART5 接收控制帧

接收长度固定为 `11` 字节：

| 字节序号 | 含义 |
| --- | --- |
| `0` | 帧头 `0x7B` |
| `1` | 预留/当前代码未使用 |
| `2` | 预留/当前代码未使用 |
| `3~4` | `X_speed`，高字节在前，单位 `mm/s` |
| `5~6` | `Y_speed`，高字节在前，单位 `mm/s` |
| `7~8` | `Z_speed`，高字节在前 |
| `9` | 前 9 字节异或校验 |
| `10` | 帧尾 `0x7D` |

说明：

- 阿克曼模式下，接收到的 `Z_speed` 会先转换成前轮转角，再做控制。

### 6.2 状态上报帧

发送长度固定为 `24` 字节，帧头帧尾同样是 `0x7B/0x7D`：

| 字节序号 | 含义 |
| --- | --- |
| `0` | 帧头 |
| `1` | 软件失能标志 `Flag_Stop` |
| `2~7` | `X/Y/Z` 速度 |
| `8~13` | IMU 加速度 `X/Y/Z` |
| `14~19` | IMU 角速度 `X/Y/Z` |
| `20~21` | 电池电压，放大 1000 倍 |
| `22` | 异或校验 |
| `23` | 帧尾 |

### 6.3 CAN 协议

| 方向 | ID | 内容 |
| --- | --- | --- |
| 接收 | `0x181` | 控制命令，前 6 字节分别为 `X/Y/Z` |
| 发送 | `0x101` | 状态帧前 8 字节 |
| 发送 | `0x102` | 状态帧中间 8 字节 |
| 发送 | `0x103` | 状态帧最后 8 字节 |

## 7. 源码备注与注意事项

### 7.1 `TIM8` 资源复用

`PC6~PC9` 既可以做航模接收输入，也可以做 `TIM8` 舵机 PWM 输出。当前工程实际启用的是“航模输入捕获”，不是“舵机输出”。

### 7.2 编码器命名存在不一致

编码器注释和控制逻辑对 `A/B/C/D` 的命名有交叉，手工接线时一定要做实测确认，不建议只凭变量名下结论。

### 7.3 `UART5` 控制路径当前基本不可用

`UART5_IRQHandler()` 里有一段保护逻辑：

- `if(Time_count < CONTROL_DELAY) return 0;`

但我在当前工程里没有找到 `Time_count++` 的实际累加代码，因此 `Time_count` 一直是 `0`，这会导致 `UART5` 接收到的数据始终被直接丢弃。

### 7.4 工程名与芯片型号存在一处不一致

- 文件夹名称写的是 `STM32F407VET6`
- 但 `Project.uvprojx` 的目标器件配置是 `STM32F407ZG`

如果你后续要重新画板或复查原理图，建议先确认你手上的实际芯片封装和 Keil 目标配置是否一致。

## 8. 建议的接线/调试顺序

1. 先接 `PB0/PB1`、`PA12`、`PE0`、`PD3`，确认电压采样、按键、使能、LED 都正常。
2. 再接 `PB10/PB11` 的 MPU6050，与 `PD11~PD14` 的 OLED，确认传感器和显示工作正常。
3. 再接一组电机 PWM 与对应编码器，先单轮验证方向。
4. 四轮全部正常后，再接 PS2、蓝牙、航模接收机、CAN 或 ROS 串口。
5. 如果是阿克曼底盘，最后再接 `PB15` 舵机，并确认中值 `1500` 是否对准机械中心。
