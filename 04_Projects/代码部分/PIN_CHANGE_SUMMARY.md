# STM32 引脚修改总结

## 当前引脚映射

| 模块 | 信号 | 新引脚 |
| --- | --- | --- |
| 五路循迹 | OUT1 ~ OUT5 | `PA2` `PA3` `PA4` `PB4` `PB5` |
| MPU6050 | `SCL` `SDA` | `PB6` `PB7` |
| OLED | `SCL` `SDA` | `PB6` `PB7` |
| TB6612 电机 A | `PWMA` `AIN1` `AIN2` | `PA8` `PB0` `PB1` |
| TB6612 电机 B | `PWMB` `BIN1` `BIN2` | `PA9` `PB10` `PB11` |
| 编码器 1 | `E1A` `E1B` | `PA6` `PA7` |
| 编码器 2 | `E2A` `E2B` | `PA0` `PA1` |
| 蜂鸣器 | `BEEP` | `PA10` |
| LED | `LED` | `PA11` |

## 已完成的代码调整

- 将循迹传感器输入从旧的 `PA4/PA5/PA12/PB0/PB1` 改成 `PA2/PA3/PA4/PB4/PB5`。
- 为了使用 `PB4`，在传感器初始化里关闭了 JTAG，仅保留 SWD 调试。
- 将 `MPU6050` 的软件 I2C 从 `PB3/PB4` 改成 `PB6/PB7`。
- 将 `OLED` 的软件 I2C 从 `PB8/PB9` 改成 `PB6/PB7`，现在和 `MPU6050` 共用同一组 I2C 引脚。
- 将电机 PWM 从 `PA8/PA11` 改成 `PA8/PA9`，对应 `TIM1_CH1/TIM1_CH2`。
- 将 TB6612 方向引脚从旧的 `PB12~PB15` 改成 `PB0/PB1/PB10/PB11`。
- 将第二路编码器从 `TIM4 + PB6/PB7` 改成 `TIM2 + PA0/PA1`。
- 因为 `TIM2` 改给了编码器，所以控制周期中断从 `TIM2` 挪到了 `TIM4`。
- 去掉了与新接线冲突的旧按键引脚 `PA0/PA1` 和旧串口引脚 `PB10/PB11` 的实际占用，相关模块保留为空实现，避免再次占用这些引脚。
- 主函数初始化顺序已调整为“先初始化外设，再启动控制中断”，避免中断过早进入。

## 说明

- 本次修改按“保持当前项目左右轮控制语义不变”的思路处理：
  - 右轮仍对应 `TIM3 + PA6/PA7` 编码器，PWM 走 `PWMA/PA8`。
  - 左轮改为 `TIM2 + PA0/PA1` 编码器，PWM 走 `PWMB/PA9`。
- `PA10` 蜂鸣器和 `PA11` LED 目前在项目逻辑里还没有实际功能代码，本次没有新增行为，只保留为未占用状态，后续可以直接继续扩展。
