# STM32手表设计方案

## 一、硬件配置

### 1.1 硬件连接

| 外设 | 引脚 | 功能 | 备注 |
|------|------|------|------|
| OLED显示屏 SCL | PB8 | I2C时钟线 | I2C1_SCL |
| OLED显示屏 SDA | PB9 | I2C数据线 | I2C1_SDA |
| 旋转编码器 A | PB0 | 编码器A相 | TIM3_CH1 |
| 旋转编码器 B | PB10 | 编码器B相 | TIM3_CH2 |
| 旋转编码器 C | PB1 | 编码器按键 | GPIO输入 |
| LED手电筒 | PA1 | 手电筒控制 | GPIO输出，共阳极接法 |

### 1.2 硬件说明
- **OLED显示屏**: 0.96寸SSD1306，I2C接口
- **旋转编码器**: 带按键的旋转编码器，支持旋转和按下功能
- **LED**: 一端接PA1，另一端接高电平，PA1输出低电平点亮LED

### 3.2 主页功能

- **中间**: 显示当前时间（时:分:秒）
- **底部左右角**: "菜单"和"设置"两个可选项
- **选中效果**: 选中项字体周围长方形像素全部点亮
- **未选中效果**: 字体所在的像素熄灭，模拟透明效果

## 四、输入系统

### 4.1 旋转编码器功能

| 操作 | 功能 |
|------|------|
| 顺时针旋转 | 切换到下一个选项 |
| 逆时针旋转 | 切换到上一个选项 |
| 按下 | 进入选中项 / 确认 |

### 4.2 编码器引脚配置
- A相 (PB0): TIM3_CH1，连接到定时器编码器接口
- B相 (PB10): TIM3_CH2，连接到定时器编码器接口
- C相 (PB1): GPIO输入，配置为上拉，检测按下事件

### 4.3 按键消抖
- 使用软件消抖，检测到下降沿后延时10-20ms再次确认
- 或使用定时器中断进行状态机消抖

## 七、软件实现细节

### 7.1 需要修改的驱动

#### Encoder.c - 编码器驱动修改
```c
// 当前编码器使用PA6/PA7 (TIM3_CH1/CH2)
// 需要修改为PB0/PB10
void Encoder_Init(void)
{
    // 修改时钟使能
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM3, ENABLE);
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE);  // 改为GPIOB

    // 修改GPIO配置
    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_0 | GPIO_Pin_10;  // 改为PB0, PB10
    GPIO_Init(GPIOB, &GPIO_InitStructure);

    // TIM3编码器接口配置...
}
```

#### 新增按键驱动 Button.c
```c
// PB1按键初始化
void Button_Init(void)
{
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE);
    GPIO_InitTypeDef GPIO_InitStructure;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPU;  // 上拉输入
    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_1;
    GPIO_Init(GPIOB, &GPIO_InitStructure);
}

// 读取按键状态（带消抖）
uint8_t Button_Get(void)
{
    if (GPIO_ReadInputDataBit(GPIOB, GPIO_Pin_1) == 0)
    {
        Delay_ms(20);  // 消抖
        if (GPIO_ReadInputDataBit(GPIOB, GPIO_Pin_1) == 0)
        {
            while (GPIO_ReadInputDataBit(GPIOB, GPIO_Pin_1) == 0);  // 等待松开
            return 1;
        }
    }
    return 0;
}
```

## 九、注意事项

1. **编码器引脚冲突**: PB0同时也是BOOT0引脚，确保外部电路不会影响系统启动
2. **LED接法**: LED接高电平，PA1输出低电平点亮，输出高电平熄灭
3. **消抖处理**: 编码器和按键都需要消抖处理
4. **显示刷新**: 避免频繁刷新导致闪烁，采用局部刷新或双缓冲
5. **时间同步**: 如需长时间断电保持，考虑使用VBAT供电RTC
